# Bug Record: BUG-2026-06-26-001

## Summary

- Title: ShipFlow installer configures every eligible user by default
- Current status: `closed`
- Severity: `high`
- Owner: unassigned
- Next command: `/sg-test --retest BUG-2026-06-26-001`

## Reproduction

- Preconditions:
  - Run `sudo ./install.sh` on a server with root plus at least one regular user account
  - Default environment variables, no explicit `SHIPFLOW_INSTALL_USERS_MODE=user-list`
- Steps:
  1. Launch the ShipFlow installer from the public bootstrap or directly as root
  2. Let the install complete with default settings
  3. Inspect the home directories of the non-root user accounts

## Expected Behavior

- The installer should configure the account that launched the installation by default
- Additional users should only be configured when explicitly requested

## Observed Behavior

- The installer used to configure root and every eligible uid>=1000 account by default
- Each account received `.claude`, `.codex`, `.bashrc`, CLI installs, MCP settings, and skills links

## Evidence

- `install.sh`
- `install-shipflow.sh`
