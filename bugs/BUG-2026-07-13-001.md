# Bug Record: BUG-2026-07-13-001

## Summary

- Title: Mail Intelligence classification crashes with Avante codex provider
- Current status: `open`
- Severity: `high`
- Owner: unassigned
- Next command: `/106-sg-fix BUG-2026-07-13-001`

## Reproduction

- Preconditions:
  - Avante is configured with the ACP provider `codex`
  - Mail Intelligence contains at least one pending email
- Steps:
  1. Open Mail Intelligence with `<leader>mi`
  2. Select an email
  3. Press `a`

## Expected Behavior

- The classification action should either capture a structured result or cleanly fall back to the interactive Avante prompt.
- Neovim should not raise an uncaught callback error.

## Observed Behavior

- Neovim reported `Failed to find provider: codex` from `avante/providers/init.lua`.
- The error occurred in `lua/shipglowz/mail/ai.lua:151` while handling the scheduled callback.

## Evidence

- User-provided stack trace; raw email content was not recorded.

## Diagnosis Notes

- `codex` is an Avante ACP provider and is not present in the HTTP provider table used by `avante.llm`.
- The classification adapter indexed it without protecting the provider lookup.
