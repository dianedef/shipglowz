# Bug Record: BUG-2026-05-08-002

## Summary

- Title: Health Check can auto-enter aggressive cleanup after pressing H
- Current status: `closed`
- Severity: `high`
- Owner: Codex
- Next command: none

## Reproduction

- Preconditions:
  - ShipFlow top-level menu is running with `gum` available.
- Steps:
  1. Open `sf`.
  2. Press `h` or `H` for Health Check, especially with an extra Enter.
  3. Observe whether Health waits at its options menu or enters cleanup.

## Expected Behavior

- Health Check should show current memory/process and PM2 health information.
- It should wait for an explicit Health sub-menu key.
- Accidental Enter should not select a destructive cleanup action.

## Observed Behavior

- The operator reported that pressing `H` sent the flow directly to the aggressive cleanup/light cleanup area before the expected process view.
- The current code put `Aggressive cleanup` as the first option in a `gum filter` menu, so a stray Enter could accept that first option.

## Evidence

- `lib.sh::action_health` used `ui_choose` with seven options.
- `ui_choose` routes lists longer than five items through `ui_filter_choose`.
- `gum filter` can accept the highlighted first item on Enter.
- The first option was `Aggressive cleanup`.
- Follow-up audit checked all `ui_choose` and direct `gum choose` call sites in `lib.sh`, `menu_gum.sh`, `menu_bash.sh`, and `local/local.sh`.
