# Bug Record: BUG-2026-05-04-003

## Summary

- Title: Blacksmith remote auth callback fails without SSH tunnel
- Current status: `fix-attempted`
- Severity: `medium`
- Owner: Diane
- Next command: `/sg-test --retest BUG-2026-05-04-003`

## Reproduction

- Preconditions:
  - ShipFlow runs on a remote server.
  - Operator opens the OAuth URL in a local browser.
  - Blacksmith CLI is installed or being authenticated on the remote server.
- Steps:
  1. Open the remote ShipFlow Blacksmith setup menu.
  2. Try to connect Blacksmith by following the direct `blacksmith auth login` flow from the server.
  3. Open the generated OAuth URL in the local browser.

## Expected Behavior

- ShipFlow should guide the operator through a local SSH callback tunnel so the local browser can reach the callback listener running on the remote server.

## Observed Behavior

- The Blacksmith OAuth URL redirects to localhost and the browser reports connection refused because localhost points to the local workstation, not the remote server.

## Evidence

- Operator report in chat on 2026-05-04.
- Existing `local/mcp-login.sh` already solves the same callback routing class for remote Codex MCP OAuth.
