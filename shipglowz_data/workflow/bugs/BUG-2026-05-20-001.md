# Bug Record: BUG-2026-05-20-001

## Summary

- Title: Terminal TUI activity panel shows headings instead of task and audit entries
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: unassigned
- Next command: `/sg-test --retest BUG-2026-05-20-001`

## Reproduction

- Preconditions:
  - ShipFlow Terminal TUI is launched from `/home/claude/shipflow/tui` or via the installed alias.
  - Activity panel is active.
  - `TASKS.md` and `AUDIT_LOG.md` use Markdown headings plus tables.
- Steps:
  1. Launch the TUI.
  2. Switch to the Activity panel with `Tab`.
  3. Read the Tasks and Audits sections.

## Expected Behavior

- Tasks should show meaningful task rows from `shipglowz_data/workflow/TASKS.md` and then master task rows if space remains.
- Audits should show recent audit table entries, not the file title or intro blockquote.
- Legacy task sections should not pollute the current activity view.

## Observed Behavior

- Tasks showed document headings such as `# ShipFlow — Master Tasks` and `## Dashboard`.
- Audits showed `# Audit Log` and the introductory blockquote instead of audit rows.

## Evidence

- User report included this visible output:
  - `Tasks:` then `# ShipFlow — Master Tasks`, `## Dashboard`
  - `Audits:` then `# Audit Log`, intro text
- Post-fix smoke rendered current task rows and recent audit rows from the Markdown tables.
