# Bug Record: BUG-2026-05-04-004

## Summary

- Title: Convex-only package.json blocks Flutter dev command detection
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: unassigned
- Next command: `/sg-verify BUG-2026-05-04-004`

## Reproduction

- Preconditions:
  - A Flutter project contains `pubspec.yaml` and a `web/` directory.
  - The same project also contains `package.json`, but only non-app scripts such as `convex:dev` and `convex:deploy`.
- Steps:
  1. Start the project through ShipFlow's normal environment lifecycle.
  2. Let `detect_dev_command` inspect the project directory.
  3. Observe the command detection result.

## Expected Behavior

- ShipFlow should fall back to `pubspec.yaml` detection when `package.json` has no runnable app `dev` or `start` script.
- A Flutter Web project should get the existing Flutter Web command:
  `flutter config --enable-web >/dev/null 2>&1 || true && flutter pub get && flutter run -d web-server --web-hostname 0.0.0.0 --web-port $PORT`.

## Observed Behavior

- ShipFlow enters the `package.json` branch first.
- Because there is no supported JS framework and no exact `dev` or `start` script, the function returns no command.
- The later `pubspec.yaml` branch is never reached, so ShipFlow reports no dev command detected for a valid Flutter project.

## Evidence

- `lib.sh:3483` starts the `if [ -f "package.json" ]` branch.
- `lib.sh:3537` begins the next `elif`, so a package-only miss cannot fall through to `pubspec.yaml`.
- Operator report: package scripts are limited to Convex commands, while the project is Flutter.
