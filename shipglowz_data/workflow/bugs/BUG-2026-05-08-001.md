# Bug Record: BUG-2026-05-08-001

## Summary

- Title: ShipFlow top-level menu takes several seconds to render with Gum
- Current status: `closed`
- Severity: `high`
- Owner: Codex
- Next command: none

## Reproduction

- Preconditions:
  - ShipFlow runs on a server where `gum` is installed and the top-level menu uses `menu_gum.sh`.
- Steps:
  1. Run `sf` or `./shipglowz.sh` from a shell.
  2. Wait for the top-level menu to appear.
  3. Compare the delay with a server using the Bash fallback menu.

## Expected Behavior

- The top-level menu should appear nearly immediately after launch.
- Rendering the menu should not depend on dozens of external process launches.

## Observed Behavior

- The operator observed at least 4 seconds before the menu appeared on the current server.
- Local timing showed the Gum menu render path was the dominant cost, while prerequisite checks and cleanup were each sub-second.

## Evidence

- `time gum --version`: 0.076s
- `time bash -lc 'source ./lib.sh'`: 0.127s
- `time bash -lc 'source ./lib.sh; check_prerequisites quiet'`: 0.122s
- `time bash -lc 'source ./lib.sh; cleanup_orphan_projects >/dev/null'`: 0.251s
- `time bash -lc 'source ./lib.sh; printf x | ( source ./menu_gum.sh; _gum_run_menu "Shipflow DevServer" "" "screen" "${MAIN_MENU_ITEMS[@]}" ) >/dev/null'`: 2.594s before fix
- `time bash -lc 'printf x | ./shipglowz.sh >/tmp/shipflow-start.out'`: 2.929s before fix
