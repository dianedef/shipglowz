# Bug Record: BUG-2026-07-17-001

## Summary

- Title: Public bootstrap requires root and bypasses the local installer on Android Termux
- Current status: `fix-attempted`
- Severity: `medium`
- Owner: unassigned
- Next command: `/405-sg-prod`, then `/107-sg-test --retest BUG-2026-07-17-001` on Android Termux

## Reproduction

- Preconditions:
  - Android Termux session with `curl` and `sh`; `sudo` is not installed.
  - Public bootstrap available at `https://www.winflowz.com/shipglowz-script`.
- Steps:
  1. Run `curl -fsSL https://winflowz.com/shipglowz-script | sudo sh`.
  2. Observe that `sudo` is unavailable and the producer reports a broken/write pipe failure because the consumer never starts.
  3. Retry without `sudo`, piping the same public bootstrap to `sh`.
  4. Observe the unconditional non-root exit and recommendation to use `sudo`; `local/install.sh` is never invoked.

## Expected Behavior

- The public bootstrap detects Android Termux before enforcing server/root installation rules.
- Termux avoids `sudo` and root requirements, clones or updates ShipGlowz under the user's home, and routes to `local/install.sh`.
- On supported interactive general installs, the bootstrap offers local versus complete installation when appropriate; Termux defaults to or forces the local path.

## Observed Behavior

- The documented `sudo` pipeline cannot run because Termux does not provide `sudo` by default, and `curl` reports the closed pipe.
- Without `sudo`, the public bootstrap exits for a non-root user, recommends `sudo`, and never reaches the existing Termux-aware local installer.

## Evidence

- User-reported real Android Termux reproduction on 2026-07-17; command and visible behavior are recorded without device identifiers, credentials, headers, or private payloads.
- Supplied tool evidence: the `www.winflowz.com` endpoint returned HTTP 200; its SHA-256 prefix `89b93` differs from the local `install-shipglowz.sh` prefix `de46`.
- Public route ownership: `/home/claude/winglowz/winglowz_site/src/pages/shipglowz-script.ts`.
- Source proof path: regression-first. The old implementation failed 12 of 21 assertions; the corrected implementation passes the expanded 24-assertion suite, including a pseudo-terminal choice.

## Correction Contract

- Classification: `spec-first`.
- Canonical authority:
  - `install-shipglowz.sh` owns bootstrap behavior.
  - The Winglowz route serves that canonical behavior through a deterministic synchronization or generation mechanism; a parity test must fail when the public payload diverges.
  - Public EN/FR copy describes the same command and mode semantics as the served script.
- Entry command and mode:
  - The public default is `curl -fsSL https://www.winflowz.com/shipglowz-script | sh`, without an outer `sudo`.
  - `SHIPGLOWZ_INSTALL_MODE=local|full` is the non-interactive contract; unsupported values fail before clone, elevation, or installer execution.
  - Android Termux always selects `local` without root. An explicit incompatible `full` request on Termux fails with an actionable diagnostic rather than attempting elevation.
  - `full` is limited to supported Ubuntu/server-style hosts and requires root only for the system-install phase.
- Interactive selection:
  - Platform and explicit mode take precedence over prompting.
  - When intent remains ambiguous and a TTY exists, prompt through `/dev/tty`, never through piped script stdin.
  - When intent remains ambiguous without a TTY, fail with the two explicit mode commands instead of guessing.
- User and home ownership:
  - Resolve the invoking user and its real home before any elevation.
  - Clone or update the repository as that user under the selected user home, preserving existing dirty-tree safety.
  - `local` executes `local/install.sh` as the invoking user; `full` delegates to `cli/install.sh` with root privileges while retaining the repository owner/home contract.
- Regression-first proof:
  - Simulated Termux without `sudo` selects `local`, uses the caller home, and never reaches the root installer.
  - Explicit `local` on a supported non-Termux workstation stays non-root.
  - Explicit `full` proves the repository remains user-owned while `cli/install.sh` receives root execution; unavailable elevation fails clearly.
  - Invalid mode and non-TTY ambiguity fail before side effects.
  - Canonical/public payload parity and EN/FR command-copy checks prevent future drift.
  - After local proof, Winglowz hybrid mode requires ship and deployed endpoint confirmation before a real Android Termux retest can move the bug beyond `fix-attempted`.
