# Bug Record: BUG-2026-05-02-002

## Summary

- Title: Local SSH server prompt accepts invalid hosts and glues menu output in Termux
- Current status: `closed`
- Severity: `medium`
- Owner: unassigned
- Next command: none

## Reproduction

- Preconditions:
  - ShipFlow local menu runs from a terminal such as Termux.
  - No exact SSH alias named `jdjdjd` exists in `~/.ssh/config`.
- Steps:
  1. Open the local ShipFlow menu.
  2. Choose `c) Configurer nouveau serveur`.
  3. Enter `jdjdjd` at `Adresse IP ou host du nouveau serveur`.
  4. Use a one-key menu choice before a screen that prints a border.

## Expected Behavior

- The server field rejects invalid free-form hosts before asking for the SSH user.
- Valid targets are IPv4 addresses, dotted DNS names, exact aliases from `~/.ssh/config`, or `user@host` forms using those host rules.
- One-key menu prompts leave the cursor on a clean next line before the next screen prints.
- The SSH user paragraph keeps the first two sentences on the same output line.

## Observed Behavior

- A random single-label value such as `jdjdjd` passed the old target regex and only failed later at the SSH test.
- One-key prompt output could be glued to the next border in Termux.
- The SSH user help text forced a newline between the first and second sentence.

## Evidence

- Chat report from 2026-05-02 with the exact prompt transcript and Termux rendering note.
