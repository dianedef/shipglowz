# Bug Record: BUG-2026-05-02-003

## Summary

- Title: Local SSH setup rejects bare relative identity file names
- Current status: `closed`
- Severity: `medium`
- Owner: unassigned
- Next command: none

## Reproduction

- Preconditions:
  - The operator can connect manually with `ssh -i oracle.key ubuntu@203.0.113.10`.
  - The key is available as a bare filename from a normal SSH command context.
- Steps:
  1. Open the local ShipFlow menu.
  2. Choose `c) Configurer nouveau serveur`.
  3. Enter `203.0.113.10` and the SSH user.
  4. Enter `oracle.key` for the identity-file prompt.

## Expected Behavior

- ShipFlow accepts `oracle.key` when the key exists in a practical SSH location.
- Bare key names are resolved from the current directory first, then `~/.ssh/`, then the user's home directory.
- The saved identity path is absolute so later tunnel commands do not depend on the menu launch directory.

## Observed Behavior

- ShipFlow rejected `oracle.key` with `Clé SSH invalide ou introuvable`.
- The user could still connect manually with `ssh -i oracle.key ubuntu@203.0.113.10`.

## Evidence

- Chat report from 2026-05-02 with the exact ShipFlow prompt output and manual SSH command.
