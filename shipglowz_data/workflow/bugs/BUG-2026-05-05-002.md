# Bug Record: BUG-2026-05-05-002

## Summary

- Title: Local tunnel restart reports no active tunnel when SSH target follows -L
- Current status: `fix-attempted`
- Severity: `medium`
- Owner: unassigned
- Next command: `/sg-test --retest BUG-2026-05-05-002`

## Reproduction

- Preconditions:
  - A ShipFlow local tunnel is running through `autossh`.
  - The process args contain `-L <port>:localhost:<port>` before the SSH target.
- Steps:
  1. Run `urls`.
  2. Choose `r) Redémarrer les tunnels`.
  3. Observe the stop phase output.

## Expected Behavior

- The menu should detect and stop ShipFlow-owned `autossh` or `ssh -N -L` tunnel processes for the configured remote host.
- If no managed tunnel exists, the normal output should stay concise.
- Raw SSH process listings should only appear in explicit debug mode.

## Observed Behavior

- The menu reported `Aucun tunnel actif trouvé` even while a matching `autossh` tunnel and its `ssh` child existed.
- The menu then printed a raw `ps` listing of SSH processes, which was noisy and exposed operator-local command details.

## Evidence

- Operator-provided restart output, redacted in this record.
- Code inspection: `local/local.sh::get_tunnel_processes` required the configured remote host to appear before `-L`.
